## Overview

Flowise can be installed in multiple ways depending on your use case:

- **NPM**: Quick global installation for production or testing
- **Docker**: Containerized deployment with Docker or Docker Compose
- **Development**: Full source code setup for contributing or customization

<Note>
  **Requirements**: All installation methods require Node.js >= 18.15.0 (versions 18.15.0 to 18.x or 20.x are supported).
</Note>

---

## NPM Installation

The fastest way to get started with Flowise.

<Steps>
  <Step title="Install Node.js">
    Download and install [Node.js](https://nodejs.org/en/download) version 18.15.0 or higher.

    Verify your installation:

    ```bash
    node --version
    ```

    Expected output: `v18.15.0` or higher
  </Step>

  <Step title="Install Flowise globally">
    ```bash
    npm install -g flowise
    ```
  </Step>

  <Step title="Start Flowise">
    ```bash
    npx flowise start
    ```

    The application will be available at `http://localhost:3000`
  </Step>
</Steps>

<Tip>
  To customize the port or other settings, use environment variables. See [Environment Variables](/configuration/environment-variables) for details.
</Tip>

## Docker Installation

Run Flowise in a containerized environment.

### Docker Compose (Recommended)

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/FlowiseAI/Flowise.git
    cd Flowise
    ```
  </Step>

  <Step title="Navigate to docker folder">
    ```bash
    cd docker
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file to customize your configuration. Key variables include:

    ```bash
    PORT=3000
    DATABASE_PATH=/root/.flowise
    LOG_PATH=/root/.flowise/logs
    SECRETKEY_PATH=/root/.flowise
    BLOB_STORAGE_PATH=/root/.flowise/storage
    ```

    <Tip>
      These paths ensure your data persists across container restarts. See the full list of [environment variables](/configuration/environment-variables).
    </Tip>
  </Step>

  <Step title="Start with Docker Compose">
    ```bash
    docker compose up -d
    ```

    The `-d` flag runs containers in detached mode (background).
  </Step>

  <Step title="Access Flowise">
    Open your browser and navigate to:

    ```
    http://localhost:3000
    ```
  </Step>

  <Step title="Stop containers (when needed)">
    ```bash
    docker compose stop
    ```
  </Step>
</Steps>

### Docker Image

Run Flowise directly from Docker Hub without cloning the repository.

<CodeGroup>
```bash Pull and Run
# Run the latest Flowise image
docker run -d \
  --name flowise \
  -p 3000:3000 \
  -v ~/.flowise:/root/.flowise \
  flowiseai/flowise:latest
```

```bash Build Locally
# Clone and build the image yourself
git clone https://github.com/FlowiseAI/Flowise.git
cd Flowise
docker build --no-cache -t flowise .
```

```bash Run Local Build
# Run your locally built image
docker run -d \
  --name flowise \
  -p 3000:3000 \
  flowise
```

```bash Stop Container
# Stop the Flowise container
docker stop flowise
```
</CodeGroup>

<Note>
  The `-v ~/.flowise:/root/.flowise` flag mounts a volume to persist your data (flows, credentials, logs) on the host machine.
</Note>

## Development Setup

Set up Flowise for development, contributing, or advanced customization.

<Warning>
  This setup is for developers who want to modify Flowise's source code. For production use, we recommend the NPM or Docker installation methods.
</Warning>

### Prerequisites

<Steps>
  <Step title="Install PNPM">
    Flowise uses PNPM as its package manager:

    ```bash
    npm i -g pnpm
    ```

    <Note>
      Flowise requires pnpm version 10.26.0 or higher.
    </Note>
  </Step>
</Steps>

### Setup Instructions

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/FlowiseAI/Flowise.git
    cd Flowise
    ```
  </Step>

  <Step title="Install dependencies">
    Install all dependencies for all modules in the monorepo:

    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Build all modules">
    ```bash
    pnpm build
    ```

    <Accordion title="Troubleshooting: JavaScript heap out of memory">
      If you encounter exit code 134 (heap out of memory), increase Node.js heap size:

      <CodeGroup>
      ```bash macOS / Linux / Git Bash
      export NODE_OPTIONS="--max-old-space-size=4096"
      pnpm build
      ```

      ```powershell Windows PowerShell
      $env:NODE_OPTIONS="--max-old-space-size=4096"
      pnpm build
      ```

      ```cmd Windows CMD
      set NODE_OPTIONS=--max-old-space-size=4096
      pnpm build
      ```
      </CodeGroup>
    </Accordion>
  </Step>

  <Step title="Start the application">
    For production mode:

    ```bash
    pnpm start
    ```

    Access the application at `http://localhost:3000`
  </Step>

  <Step title="Development mode (optional)">
    For development with hot-reload:

    1. Create `.env` file in `packages/ui` and specify `VITE_PORT` (refer to `.env.example`)
    2. Create `.env` file in `packages/server` and specify `PORT` (refer to `.env.example`)
    3. Run development build:

    ```bash
    pnpm dev
    ```

    The app will be available at `http://localhost:8080` with auto-reload on code changes.
  </Step>
</Steps>

## Configuration

After installation, you can configure Flowise using environment variables:

<CodeGroup>
```bash Basic Configuration
PORT=3000
DATABASE_PATH=/your_database_path/.flowise
LOG_PATH=/your_log_path/.flowise/logs
```

```bash Database (PostgreSQL)
DATABASE_TYPE=postgres
DATABASE_PORT=5432
DATABASE_HOST=localhost
DATABASE_NAME=flowise
DATABASE_USER=root
DATABASE_PASSWORD=mypassword
```

```bash Authentication
APP_URL=http://localhost:3000
JWT_AUTH_TOKEN_SECRET=your-secret-key
JWT_REFRESH_TOKEN_SECRET=your-refresh-secret
JWT_ISSUER=Flowise
JWT_AUDIENCE=Flowise
```

```bash Storage
STORAGE_TYPE=local
BLOB_STORAGE_PATH=/your_storage_path/.flowise/storage
# Or use S3
# STORAGE_TYPE=s3
# S3_STORAGE_BUCKET_NAME=flowise
# S3_STORAGE_ACCESS_KEY_ID=your-access-key
# S3_STORAGE_SECRET_ACCESS_KEY=your-secret-key
```
</CodeGroup>

For a complete list of configuration options, see [Environment Variables](/configuration/environment-variables).
