title: Environment Variables
description: Complete reference for all Flowise environment variables
---

Flowise provides extensive configuration through environment variables. This guide covers all available options organized by category.

## Server Configuration

<ParamField path="PORT" type="number" default="3000">
  Port number for the Flowise server to listen on
</ParamField>

<ParamField path="APP_URL" type="string" default="http://localhost:3000">
  Base URL of your Flowise application, used for generating links in emails and SSO callbacks
</ParamField>

<ParamField path="NUMBER_OF_PROXIES" type="number" default="1">
  Number of proxies between client and server. Used for proper IP address resolution
</ParamField>

<ParamField path="TRUST_PROXY" type="string" default="false">
  Trust proxy configuration for Express. Can be `true`, `false`, `1`, `loopback`, `linklocal`, `uniquelocal`, IP addresses, or comma-separated combinations
</ParamField>

<ParamField path="CORS_ORIGINS" type="string" default="*">
  Comma-separated list of allowed CORS origins. Use `*` to allow all origins
</ParamField>

<ParamField path="IFRAME_ORIGINS" type="string" default="*">
  Comma-separated list of allowed iframe origins. Use `*` to allow all origins
</ParamField>

<ParamField path="FLOWISE_FILE_SIZE_LIMIT" type="string" default="50mb">
  Maximum file upload size (e.g., `50mb`, `100mb`)
</ParamField>

## Database Configuration

Flowise supports SQLite (default), PostgreSQL, MySQL, and MariaDB.

<ParamField path="DATABASE_TYPE" type="string" default="sqlite">
  Database type. Options: `sqlite`, `postgres`, `mysql`, `mariadb`
</ParamField>

<ParamField path="DATABASE_PATH" type="string" default="~/.flowise">
  Path where the SQLite database file will be stored. Only used when `DATABASE_TYPE=sqlite`
</ParamField>

### PostgreSQL / MySQL / MariaDB Configuration

<ParamField path="DATABASE_HOST" type="string">
  Database server hostname or IP address
</ParamField>

<ParamField path="DATABASE_PORT" type="number" default="5432 (postgres) | 3306 (mysql/mariadb)">
  Database server port
</ParamField>

<ParamField path="DATABASE_NAME" type="string" default="flowise">
  Name of the database to use
</ParamField>

<ParamField path="DATABASE_USER" type="string" default="root">
  Database username
</ParamField>

<ParamField path="DATABASE_PASSWORD" type="string">
  Database password
</ParamField>

### Database SSL Configuration

<ParamField path="DATABASE_SSL" type="boolean" default="false">
  Enable SSL connection to database
</ParamField>

<ParamField path="DATABASE_REJECT_UNAUTHORIZED" type="boolean" default="true">
  Whether to reject unauthorized SSL certificates
</ParamField>

<ParamField path="DATABASE_SSL_KEY_BASE64" type="string">
  Self-signed SSL certificate in BASE64 format for database connection
</ParamField>

## Authentication & Security

### Basic Authentication

<ParamField path="FLOWISE_USERNAME" type="string">
  Username for basic HTTP authentication (legacy)
</ParamField>

<ParamField path="FLOWISE_PASSWORD" type="string">
  Password for basic HTTP authentication (legacy)
</ParamField>

### JWT Configuration

<ParamField path="JWT_AUTH_TOKEN_SECRET" type="string">
  Secret key for signing JWT access tokens. Generate with: `openssl rand -hex 32`
  
  If not set, uses file-based or AWS Secrets Manager storage
</ParamField>

<ParamField path="JWT_REFRESH_TOKEN_SECRET" type="string">
  Secret key for signing JWT refresh tokens. Generate with: `openssl rand -hex 32`
  
  If not set, uses file-based or AWS Secrets Manager storage
</ParamField>

<ParamField path="JWT_ISSUER" type="string" default="Flowise">
  JWT issuer claim value
</ParamField>

<ParamField path="JWT_AUDIENCE" type="string" default="Flowise">
  JWT audience claim value
</ParamField>

<ParamField path="JWT_TOKEN_EXPIRY_IN_MINUTES" type="number" default="360">
  Access token expiration time in minutes (default: 6 hours)
</ParamField>

<ParamField path="JWT_REFRESH_TOKEN_EXPIRY_IN_MINUTES" type="number" default="43200">
  Refresh token expiration time in minutes (default: 30 days)
</ParamField>

<ParamField path="EXPIRE_AUTH_TOKENS_ON_RESTART" type="boolean" default="false">
  Expire all authentication tokens when the application restarts
</ParamField>

### Session & Token Security

<ParamField path="EXPRESS_SESSION_SECRET" type="string">
  Secret for Express session signing. Generate with: `openssl rand -hex 32`
  
  If not set, uses file-based or AWS Secrets Manager storage
</ParamField>

<ParamField path="TOKEN_HASH_SECRET" type="string">
  Secret for hashing tokens. Generate with: `openssl rand -hex 32`
  
  If not set, uses file-based or AWS Secrets Manager storage
</ParamField>

<ParamField path="SECURE_COOKIES" type="boolean">
  Enable secure flag on cookies (recommended for HTTPS deployments)
</ParamField>

<ParamField path="INVITE_TOKEN_EXPIRY_IN_HOURS" type="number" default="24">
  Workspace invitation token expiration time in hours
</ParamField>

<ParamField path="PASSWORD_RESET_TOKEN_EXPIRY_IN_MINS" type="number" default="15">
  Password reset token expiration time in minutes
</ParamField>

<ParamField path="PASSWORD_SALT_HASH_ROUNDS" type="number" default="10">
  Number of bcrypt salt rounds for password hashing
</ParamField>

## Secret Key Storage

<ParamField path="SECRETKEY_STORAGE_TYPE" type="string" default="local">
  Storage type for encryption keys and auth secrets. Options: `local`, `aws`
</ParamField>

<ParamField path="SECRETKEY_PATH" type="string" default="~/.flowise">
  Local filesystem path for storing encryption keys and auth secrets when using `local` storage
</ParamField>

<ParamField path="FLOWISE_SECRETKEY_OVERWRITE" type="string">
  Override the encryption key with a specific value (not recommended for production)
</ParamField>

### AWS Secrets Manager Configuration

Required when `SECRETKEY_STORAGE_TYPE=aws`:

<ParamField path="SECRETKEY_AWS_ACCESS_KEY" type="string">
  AWS access key ID for Secrets Manager
</ParamField>

<ParamField path="SECRETKEY_AWS_SECRET_KEY" type="string">
  AWS secret access key for Secrets Manager
</ParamField>

<ParamField path="SECRETKEY_AWS_REGION" type="string" default="us-west-2">
  AWS region for Secrets Manager
</ParamField>

<ParamField path="SECRETKEY_AWS_NAME" type="string" default="FlowiseEncryptionKey">
  Name of the secret in AWS Secrets Manager for the main encryption key
</ParamField>

<ParamField path="SECRETKEY_AWS_AUTH_PREFIX" type="string" default="Flowise">
  Prefix for auth secret names in AWS Secrets Manager (e.g., `FlowiseTokenHashSecret`)
</ParamField>

  Comma-separated list of HTTP header names to sanitize in logs
</ParamField>

## Example Configuration

```bash
# Server
PORT=3000
APP_URL=https://flowise.yourdomain.com

# Database (PostgreSQL)
DATABASE_TYPE=postgres
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=flowise
DATABASE_USER=flowise_user
DATABASE_PASSWORD=secure_password
DATABASE_SSL=true

# JWT Authentication
JWT_AUTH_TOKEN_SECRET=$(openssl rand -hex 32)
JWT_REFRESH_TOKEN_SECRET=$(openssl rand -hex 32)
JWT_TOKEN_EXPIRY_IN_MINUTES=360

# Storage (S3)
STORAGE_TYPE=s3
S3_STORAGE_BUCKET_NAME=my-flowise-bucket
S3_STORAGE_REGION=us-east-1

# Email
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=noreply@yourdomain.com
SMTP_PASSWORD=app_specific_password
SENDER_EMAIL=team@yourdomain.com

# Logging
LOG_LEVEL=info
DEBUG=false
```

<Note>
For production deployments, always generate secure random values for secret keys using `openssl rand -hex 32`
</Note>

<Warning>
Never commit `.env` files containing secrets to version control. Use environment-specific configuration and secret management tools.
</Warning>
